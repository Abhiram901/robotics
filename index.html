// SmartSys — Arduino Multi-Function System
#include <LiquidCrystal_I2C.h>
#include <Servo.h>

LiquidCrystal_I2C lcd(0x27, 16, 2);
Servo door;

// Pins
int ledPin    = 13;   // Task 1 – LED Output
int ldrPin    = A0;   // Task 6 – LDR Light Sensor
int tempPin   = A1;   // Task 7 – Temperature Sensor
int buttonPin = 2;    // Task 3 – Button Input
int buzzer    = 8;    // Task 4 – Buzzer Alert
// Servo on pin 9  → Task 8 – Servo Motor
// LCD I2C        → Task 5 – Display

bool doorOpen = false;

void setup() {
  pinMode(ledPin,    OUTPUT);
  pinMode(buttonPin, INPUT_PULLUP);
  pinMode(buzzer,    OUTPUT);
  door.attach(9);
  lcd.init(); lcd.backlight();
  lcd.print("Smart System");
  delay(2000); lcd.clear();
}

void loop() {
  // 🌙 LDR → LED auto control
  int light = analogRead(ldrPin);
  digitalWrite(ledPin, light < 500 ? HIGH : LOW);

  // 🌡️ TMP36 → Temperature
  float tempC = analogRead(tempPin) * (5.0/1023.0) * 100;

  // 🚨 Buzzer alert if temp > 35°C
  tempC > 35 ? tone(buzzer, 1000) : noTone(buzzer);

  // 🔘 Button toggles door servo
  if (digitalRead(buttonPin) == LOW) { doorOpen = !doorOpen; delay(300); }
  door.write(doorOpen ? 90 : 0);

  // 📟 Update LCD
  lcd.setCursor(0,0); lcd.print("Temp:"); lcd.print(tempC); lcd.print("C ");
  lcd.setCursor(0,1); lcd.print("Light:"); lcd.print(light);
  delay(500);
}

System Overview

8 Embedded
System Tasks

Each task maps to a real-world embedded use case, all unified in a single Arduino program.

TASK 01 / 08
💡
Basic Output Control

An LED connected to pin 13 acts as an automatic indicator light, turning on or off based on system logic.

LED Pin 13 digitalWrite
Implementation

LED activates automatically when the LDR detects darkness (light < 500), mirroring street lighting and indicator panel systems.

TASK 02 / 08
🎚️
Variable Control System

Analog sensor values are continuously read and mapped to dynamic system behaviors — no fixed on/off states.

analogRead LDR TMP36 A0 / A1
Implementation

Both the LDR and TMP36 produce variable 0–1023 analog values, enabling gradient-based control of the LED and buzzer threshold.

TASK 03 / 08
🔘
User Interaction System

A physical pushbutton lets the user manually override the door/servo position through toggle logic.

Button Pin 2 INPUT_PULLUP Toggle
Implementation

Button on pin 2 with INPUT_PULLUP. Each LOW press toggles doorOpen, simulating an access control panel.

TASK 04 / 08
🚨
Alert / Notification System

A piezo buzzer triggers a continuous 1kHz tone whenever the temperature exceeds a safe threshold.

Buzzer Pin 8 tone() 1000 Hz
Implementation

If tempC > 35, tone(buzzer, 1000) fires — an industrial overheat alert modelled in miniature.

TASK 05 / 08
📟
Display / Information System

A 16×2 I2C LCD displays live temperature and light level readings — a real-time monitoring dashboard.

LCD I2C 0x27 16×2 LiquidCrystal
Implementation

Row 0 shows Temp: XX.X C, Row 1 shows Light: XXXX, updated every 500 ms.

TASK 06 / 08
☀️
Environmental Sensing

An LDR photoresistor continuously senses ambient light levels, enabling automatic responses to lighting conditions.

LDR A0 Photoresistor Smart City
Implementation

Analog pin A0 reads LDR resistance changes as voltage — the foundation of smart street lights and energy-saving automation.

TASK 07 / 08
🌡️
Monitoring System

A TMP36 temperature sensor provides continuous thermal monitoring with voltage-to-Celsius conversion.

TMP36 A1 5V Reference Thermal
Implementation

Formula: V × 100 = °C where V = ADC × 5/1023. Perfect for smart home / industrial HVAC control.

TASK 08 / 08
⚙️
Motion / Actuation System

A servo motor rotates between 0° and 90° to physically open or close a door mechanism.

Servo Pin 9 PWM 0–90°
Implementation

door.write(90) opens, door.write(0) closes — mimicking automated gate, lock, or robotic arm positioning.
